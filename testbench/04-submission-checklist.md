# 04 - Submission Checklist (Round 1)

## Required Deliverables
- [ ] Public GitHub repository link
- [ ] Complete `README.md` with setup and run steps
- [ ] `testbench/` folder present with grader runbook
- [ ] 1-2 minute demo video link (public or unlisted)
- [ ] Project documentation PDF (methodology, results, testing, findings)

## Repository Readiness
- [ ] `.env` is not committed
- [ ] `.env.example` is complete and current
- [ ] Secrets rotated if ever exposed
- [ ] Main branch and hosting branch in intended final state

## Demo Readiness
- [ ] Login/signup works
- [ ] No anonymous usage path
- [ ] Practice analyze works (text + file path)
- [ ] Quiz generation works with selected source
- [ ] Flashcard generation respects requested count
- [ ] Tutor response is coherent and stable
- [ ] RAG index/query path demonstrated

## Verification Artifacts to Capture
- [ ] `/api/health` screenshot (services configured)
- [ ] Auth flow screenshots
- [ ] Practice analyze + quiz + flashcard screenshots
- [ ] Tutor interaction screenshots
- [ ] Optional short log excerpt for API calls

## Honesty and Claims
- [ ] All claims in PDF are verifiable from code
- [ ] Limitations are explicitly documented
- [ ] External resources/APIs cited in documentation
