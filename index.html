7.1 Introduction to Dynamic Programming

Estimated read: 8 min · 5 paragraphs

✓

Dynamic Programming (DP) is a method for solving complex problems by breaking them down into simpler subproblems. It is applicable when subproblems overlap and the solution to each subproblem can be computed once and stored for future use, avoiding redundant computations. This technique was pioneered by Richard Bellman in the 1950s.

🤖 SpeedUp AI · Explanation

Think of DP like a smart notebook. Instead of solving the same math problem over and over (like Fibonacci), you write down the answer the first time and look it up later.

Real example: Finding the cheapest flight route. Instead of recalculating every possible path from scratch, DP stores the cheapest known cost to each intermediate city.

📌 Connected to: Memoization, Recursion (Ch.5), Bellman-Ford Algorithm
+

The two key properties that make a problem suitable for dynamic programming are optimal substructure and overlapping subproblems. Optimal substructure means the optimal solution to the problem can be constructed from optimal solutions of its subproblems. Overlapping subproblems means the same subproblems are encountered multiple times during the computation.

+

There are two main approaches to implementing dynamic programming: top-down with memoization and bottom-up tabulation. In the top-down approach, the problem is solved recursively, but results of subproblems are cached (memoized). In the bottom-up approach, subproblems are solved iteratively, starting from the smallest and building up.

+

The time complexity of DP solutions depends on the number of unique subproblems and the time taken to solve each. For instance, the classic 0/1 Knapsack problem has O(nW) complexity where n is the number of items and W is the maximum weight capacity. Space complexity can often be optimized by only storing relevant rows of the DP table.

+

Classic DP problems include the Fibonacci sequence, Longest Common Subsequence (LCS), Matrix Chain Multiplication, Coin Change problem, and Shortest Path algorithms like Bellman-Ford. Mastering these canonical problems provides a framework for recognizing and solving new DP problems in competitive programming and technical interviews.

Think of a recipe book:

🗒 Memoization (Top-Down) = You start cooking a dish. Every time you figure out a sub-step, you write it in the book. Next time you need that step, you look it up instead of re-figuring.

📋 Tabulation (Bottom-Up) = Before cooking anything, you fill in every possible sub-step in a big table, then use that table when you need to cook.

Key difference: Memo only computes what's needed. Tabulation computes everything upfront.
