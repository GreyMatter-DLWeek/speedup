AI Architecture — How It Works
1. Knowledge State Modeling
Uses Bayesian Knowledge Tracing (BKT) per topic to model your evolving mastery. Updates after every question attempt and reading session.
2. Pattern Detection Agent
Analyzes error patterns across sessions to distinguish genuine knowledge gaps from careless errors using attempt history and time signals.
3. Recommendation Agent
Generates personalized study plans based on mastery gaps, exam proximity, time availability, and your historically effective study patterns.
4. Evaluator Loop (Maker-Checker)
Each recommendation is checked for bias, fairness, and consistency before display. The student's disagree feedback is used to improve future recommendations.
Bias & Fairness Commitment
⚖️
Equitable Treatment
Recommendations do not vary based on demographic factors. The AI only uses your individual interaction data to generate personalized guidance.
📊
Consistency
The same learning data will always produce the same recommendations. AI outputs are deterministic and auditable, not random.
🗑
Data Minimization
SpeedUp only stores what's needed: attempt counts, timestamps, mastery scores, and topic tags. No keystroke logging, no screen recording.
